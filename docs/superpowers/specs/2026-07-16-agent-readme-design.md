# EssayWeb Agent README 设计

## 目标

将仓库根目录 `README.md` 改写为一份“人能快速理解、Agent 能直接执行”的项目手册。下次 Agent 只读取 README 和其中指向的实时源文件，即可在不猜测、不破坏现有内容的前提下完成论文导入、网站同步、验证与公开部署。

## 读者与优先级

README 同时服务两类读者：

1. 普通访客：快速了解网站用途、公开地址、技术栈和本地启动方式。
2. 执行论文发布的 Agent：获得完整、明确、可验证的操作规范。

当两类内容发生篇幅冲突时，优先保证 Agent 工作流完整；项目介绍保持简洁。

## README 结构

1. 项目简介与公开网站地址。
2. 当前数据模型：`D:\essay` 是唯一内容源，`D:\essayweb` 只保存静态快照和页面代码。
3. Agent 快速入口：收到标题、arXiv、DOI、项目页、网页或 PDF 时应如何处理。
4. 完整论文导入流程：来源核验、去重、完整中文笔记、源库同步、网站快照、测试、提交与部署。
5. 源库同步面：论文 Markdown、12 列 CSV、README、topic、TERMS，以及非默认领域使用的 `paper-fields.csv`。
6. 五个网站领域及 ID，说明未显式映射的论文默认进入具身智能，禁止仅凭关键词跨区猜测。
7. 阻断条件与降级规则：来源不足、重复冲突、脏工作区、CSV 或链接校验失败时停止；不得伪造全文、PDF、代码或实验数据。
8. 实际命令：同步、类型检查、单元测试、浏览器测试、构建和本地预览。
9. 发布规则：全部验证通过后直接推送 `main`，等待 GitHub Pages 成功，并验证公开论文页面返回 HTTP 200 且标题正确；只有用户明确要求时才创建 PR。
10. 代码与数据入口说明、GitHub Pages 工作流说明。

## Agent 数据流

论文来源首先写入 `D:\essay` 的各同步面；随后执行 `npm run sync:essay` 生成 `src/data/generated/library.json`。网站页面只读取生成快照，GitHub Actions 不访问本机 D 盘，也不复制本地 PDF。

## 安全与一致性

- 不修改或删除无关文件，不覆盖用户已有改动。
- 新论文默认状态为“已导入待精读”，不冒充用户已经阅读。
- 标题输入必须通过完整标题精确搜索和权威来源交叉核验；无法唯一确定时向用户索要链接或文件。
- 来源不足以完成完整笔记时停止，不发布简版占位内容。
- 保留 `Coming Soon` 等可用性限定，不把计划发布描述为已开放。
- 本地绝对路径和本地 PDF 不得进入公开快照。
- README 中不写死会频繁过期的论文、主题和术语数量；需要数量时从生成快照或同步命令实时读取。

## 验收标准

- README 中包含从输入论文到公开页面验证的完整闭环。
- 所有命令、路径、脚本名和领域 ID 与当前仓库一致。
- 明确区分源库内容和网站生成数据，Agent 不会直接编辑快照。
- 明确默认直接部署 `main`，且部署失败时不得声称发布成功。
- Markdown 链接有效，文本不存在 `TODO`、`TBD` 或相互矛盾的规则。
